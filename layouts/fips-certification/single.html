{{/* 3. What is FIPS? / Why it matters (two columns) */}}

{{ i18n "What is FIPS?" }}

{{ i18n "FIPS (Federal Information Processing Standards) are cryptographic standards published by the United States government. FIPS 140-3 is the current version, and it defines the security requirements a cryptographic module must meet." }}

{{ i18n "A module earns validation only after an accredited laboratory tests it and the NIST Cryptographic Module Validation Program certifies the result. AlmaLinux OS 9.2 was the first software implementation to receive a FIPS 140-3 ESV certificate using SHA3-256 as a conditioner." }}

{{ i18n "Why it matters" }}

{{ i18n "FIPS 140-3 is recognized and respected worldwide. The European Union's NIS 2 Directive, for example, calls for up-to-date encryption built on established standards." }}

{{ i18n "For government, finance, healthcare, and anyone handling sensitive data, running on validated cryptography is often a requirement rather than a nice to have. FIPS validation gives those organizations independent assurance about the cryptography underneath their systems." }}

{{ i18n "Validated cryptography is also a prerequisite for the DISA Security Technical Implementation Guide (STIG). AlmaLinux OS 9 is one of the few RHEL family distributions with a published DISA STIG, which gives regulated and United States Department of Defense environments an official hardening baseline. See the" }} {{ i18n "AlmaLinux OS 9 STIG on the NIST National Checklist Program" }} {{ i18n "and TuxCare's" }} {{ i18n "FIPS and STIG overview" }}.

{{/* 4. Enable FIPS mode (practical steps) + the 9.2 support caveat */}}

{{ i18n "Enable FIPS mode for free" }}

{{ i18n "Anyone running AlmaLinux OS 9.2 can enable FIPS mode and meet the standard at no cost. Install the packages from the TuxCare repository, turn on FIPS mode, and reboot:" }} 

dnf -y install https://repo.tuxcare.com/fips/tuxcare-fips-release-latest-9.noarch.rpm
dnf -y install openssl-3.0.7-20.el9_2.tuxcare.1 kernel-5.14.0-284.11.1.el9_2.tuxcare.5
fips-mode-setup --enable
reboot

{{ i18n "Once the system comes back up it is running validated cryptography. For a full walkthrough, including how to verify it, see the" }} {{ i18n "FIPS validation update blog post" }}.
