{{/* 4. How we respond to security issues */}}

{{ i18n "How we respond to security issues" }}

{{ i18n "The volume of disclosed vulnerabilities keeps rising, and AI-assisted research, proof-of-concept creation, and public disclosure are accelerating it further. No distribution can promise to fix every vulnerability on every timeline. What we can offer is a clear, consistent process for how we handle them." }}

{{ i18n "Patching from upstream" }}

{{ i18n "We follow upstream for most patches. AlmaLinux OS is built from the same sources as the rest of the enterprise Linux ecosystem, so as upstream fixes land we build, test, and publish them. Each security fix ships as an advisory (an ALSA), rated Critical, Important, Moderate, or Low, with machine-readable OVAL and OSV data and an announcement to the security mailing list." }}

  • {{ i18n "AlmaLinux Errata" }}
  • {{ i18n "AlmaLinux Errata Wiki page" }}

{{ i18n "Patching ahead of upstream" }}

{{ i18n "Sometimes an issue matters enough to the community that we apply a patch ahead of upstream. These are reviewed and approved by ALESCo, the AlmaLinux Engineering Steering Committee, which guides the technical direction of the distribution. When upstream ships its own fix, we re-align with it." }} {{ i18n "Whenever possible or appropriate, we also send patches that we've released upstream: everyone should benefit from the work of open source." }} {{ i18n "Anyone can request that a patch be considered by raising it in the" }} {{ i18n "ALESCo channel" }} {{ i18n "on chat.almalinux.org. A few examples of patches we have shipped or provided for testing:" }}

  • {{ i18n "XZ backdoor and CVE-2024-1086" }}
  • {{ i18n "regreSSHion (CVE-2024-6387)" }}
  • {{ i18n "CVE-2025-27363 patches" }}

{{ i18n "Direct reports" }}

{{ i18n "How we handle an issue reported directly to us depends on what it affects. We aim to acknowledge reports within 2 to 3 days." }}

  • {{ i18n "OS issues that need coordinated disclosure: email security@almalinux.org so we can coordinate a responsible patch and release." }}
  • {{ i18n "OS issues that do not need coordination: file them at bugs.almalinux.org." }}
  • {{ i18n "Anything that is not the operating system itself, such as ELevate or this website: open an issue on that project's repository." }}

{{ i18n "If a direct report is really an upstream issue, we point you to report it upstream so it is fixed at the source for everyone. Critical issues reported to the linux-distros mailing list are patched on the date of disclosure. See our" }} {{ i18n "vulnerability disclosure policy" }} {{ i18n "for full details." }}
