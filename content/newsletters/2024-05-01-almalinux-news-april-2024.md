Hello everyone! Welcome to this month's AlmaLinux newsletter, where we share the latest AlmaLinux happenings, upcoming events, and more.

## Quick Notes

- AlmaLinux 9.4 and 8.10 will be released in May! We appreciate everyone who is helping us test out the 9.4 Beta and the 8.10 Beta. 8.10 ISO images and repos will be used until 2029, so help us test now to make sure they're perfect!
- RHEL 8.10 and 9.4 are on their way, and Red Hat has made changes to the way they handle device drivers that are deprecated, disabled and unmaintained, and ELRepo will no longer be able to enable support for devices that Red Hat has deprecated.
- To help the community, both AlmaLinux 9.4 and 8.10 have re-enabled support for those PCI IDs, continuing support for more than 140 devices.
- Some changes have been made to the mirror system to help better manage hardware resources. If you host one of the 350+ AlmaLinux mirrors, check the forums post about the vault rsync URL change.
- The recordings of AlmaLinux Day: Germany event are now live on YouTube.

## Latest from the AlmaLinux Blog

**Why I wanted to help translate the AlmaLinux website to Hebrew**

Community member Noam Alum is helping translate the AlmaLinux website into Hebrew and explains his motivation in a recent blog post.

**AlmaLinux 8.10 Beta and 9.4 Beta**

Both versions — "Cerulean Leopard" (8.10) and "Seafoam Ocelot" (9.4) — are now available for all supported architectures. Testing is encouraged to identify bugs early.

**Secure Boot with AlmaLinux**

Enabling Secure Boot is one of the easiest ways to enhance system security. AlmaLinux has provided Secure Boot support since version 8.4. A detailed guide explains implementation and importance.

**CVE-2024-1086 and XZ**

Patches for CVE-2024-1086 became available in production repositories as of April 3rd.
