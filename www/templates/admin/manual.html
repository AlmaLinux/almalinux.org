Canonical names (slugs), languages

Website content, such as pages and blog posts have a canonical name, often called "slug". Slugs are used to provide an unique URL for each content item. Slugs are unique for each content item and do not change between languages.

Slugs are NOT unique - each content class (blog, page) can have one or more items marked with the same slug.

For example, the same slug can be used for the same content class with different content language to provide localization for content.

It is also possible to temporarily replace content item by setting a publication date after the publication date of an existing item to either temporarily or permanently override content for the same slug.

Multilingual content resolution

All content is, by default, loaded using the locale the visitor has chosen. If the content is not available, for example, a page, etc. using the users locale, English language content will be used instead.
